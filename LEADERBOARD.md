# StreamArena Baseline Leaderboard

Generated by `benchmarks/aggregate.py` from `benchmarks/results/`. Evaluation protocol (prequential test-then-train, seeds, default hyperparameters): see [BENCHMARK.md](BENCHMARK.md).

## How to read this leaderboard

- **Avg rank** — the leaderboard ordering. On every dataset the algorithms are ranked 1st, 2nd, 3rd, … on that task's primary metric (seed-averaged), and those per-dataset ranks are averaged. Lower is better; 1.00 would mean first place on every dataset. We rank instead of averaging raw metric values because raw values are not comparable across datasets (different scales, class balances, and difficulty).
- **Wins** — number of datasets where the algorithm ranked 1st.
- **Median κt (kappa-temporal)** — classification skill relative to the NoChange baseline that always predicts the previous label: 100 is perfect, 0 means no better than NoChange, negative means worse. On temporally correlated streams plain accuracy is inflated, so κt is the honest streaming metric. Median (not mean) because κt is unbounded below on near-constant streams.
- **Mean acc** — plain prequential accuracy, for intuition only.
- **Median R²** — regression fit quality, scale-free across datasets (1 = perfect, 0 = no better than predicting the mean).
- **Mean ARI** — clustering agreement with ground-truth labels, averaged over evaluation windows (1 = perfect, 0 = random).
- **Mean AUC** — anomaly-detection ROC-AUC over the full stream of scores (1 = perfect, 0.5 = random).
- **#Datasets** — coverage; algorithms with documented coverage caps (see BENCHMARK.md) run on fewer datasets and their avg rank is computed only where they ran.
- **Total wallclock (h)** — total compute across all runs (all datasets and seeds). Read it against quality: a small rank gap for a 100× time gap matters in streaming settings.
